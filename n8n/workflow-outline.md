# Workflow Outline: Inbound Lead Qualification Automation

This document describes every node in the n8n workflow — its configuration, inputs, outputs, and how it connects to the next step. Use it as a build guide when setting up the workflow in n8n.

---

## Node 1 — Inbound Lead Webhook

**Type:** Webhook  
**Purpose:** Entry point. Receives a JSON POST from a lead form, a Typeform webhook, or a direct API call.

**Configuration:**
- HTTP Method: `POST`
- Path: `/inbound-lead`
- Response Mode: `Immediately` (returns 200 OK on receipt)
- Authentication: None (add Header Auth for production)

**Expected Input:** See `test-webhook-payload.json`

**Output fields passed forward:**
```
lead_id, first_name, last_name, company, job_title,
company_size, industry, pain_point, budget_range,
timeline, source, submitted_at,
qualification.lead_score, qualification.fit_level,
qualification.qualification_reason,
qualification.recommended_next_action,
qualification.draft_follow_up_email.*
```

**Notes:** In a production setup, the qualification object would be generated by calling a Python API endpoint before this step, or by a Code node that runs scoring logic inline. For this demo, qualification is included in the incoming payload.

## Node 2 — Format Lead Fields (Set Node)

**Type:** Set  
**Purpose:** Normalizes field names, flattens nested qualification fields, and adds a `received_at` timestamp so downstream nodes work with a flat, consistent record.

**Configuration — fields to set:**

| Field Name | Value Expression |
|------------|-----------------|
| `lead_id` | `{{ $json.lead_id }}` |
| `full_name` | `{{ $json.first_name }} {{ $json.last_name }}` |
| `company` | `{{ $json.company }}` |
| `job_title` | `{{ $json.job_title }}` |
| `company_size` | `{{ $json.company_size }}` |
| `industry` | `{{ $json.industry }}` |
| `pain_point` | `{{ $json.pain_point }}` |
| `budget_range` | `{{ $json.budget_range }}` |
| `timeline` | `{{ $json.timeline }}` |
| `source` | `{{ $json.source }}` |
| `lead_score` | `{{ $json.qualification.lead_score }}` |
| `fit_level` | `{{ $json.qualification.fit_level }}` |
| `qualification_reason` | `{{ $json.qualification.qualification_reason }}` |
| `next_action` | `{{ $json.qualification.recommended_next_action }}` |
| `email_subject` | `{{ $json.qualification.draft_follow_up_email.subject }}` |
| `email_body` | `{{ $json.qualification.draft_follow_up_email.body }}` |
| `email_to` | `{{ $json.qualification.draft_follow_up_email.to }}` |
| `received_at` | `{{ $now.toISO() }}` |

**Mode:** Keep Only Set

---

## Node 3 — Check Fit Level (IF Node)

**Type:** IF  
**Purpose:** Routes the lead down two paths — High-fit leads go to the priority Sheets tab and trigger Slack; all other leads go to the general Sheets tab only.

**Condition:**
```
{{ $json.fit_level }} equals (string) "High"
```

**True branch → Node 4 (Append to Priority Sheet)**  
**False branch → Node 5 (Append to All Leads Sheet)**

---

## Node 4 — Append to Sheets: High-Fit Queue (True branch)

**Type:** Google Sheets  
**Purpose:** Writes High-fit leads to a dedicated `High-Fit Queue` tab so sales can work the priority list directly from Google Sheets.

**Configuration:**
- Operation: `Append`
- Spreadsheet ID: `YOUR_SPREADSHEET_ID` (replace in n8n credential)
- Sheet Name: `High-Fit Queue`
- Column Mapping (mapped to Set node output):

| Sheet Column | n8n Field |
|---|---|
| Received At | `received_at` |
| Lead ID | `lead_id` |
| Full Name | `full_name` |
| Company | `company` |
| Job Title | `job_title` |
| Company Size | `company_size` |
| Industry | `industry` |
| Pain Point | `pain_point` |
| Budget | `budget_range` |
| Timeline | `timeline` |
| Source | `source` |
| Lead Score | `lead_score` |
| Fit Level | `fit_level` |
| Reason | `qualification_reason` |
| Next Action | `next_action` |

**Credential:** Google OAuth2 (configured in n8n, not hardcoded)

---

## Node 5 — Append to Sheets: All Leads (False branch + merge)

**Type:** Google Sheets  
**Purpose:** Writes every lead (High, Medium, and Low) to an `All Leads` tab for a complete audit trail.

**Configuration:**
- Operation: `Append`
- Spreadsheet ID: `YOUR_SPREADSHEET_ID`
- Sheet Name: `All Leads`
- Column Mapping: Same as Node 4

**Notes:** Connect both the True and False branches here — High-fit leads write to both sheets, all others write only to All Leads.

---

## Node 6 — Create Gmail Draft

**Type:** Gmail  
**Purpose:** Creates a draft follow-up email in the connected Gmail inbox so the sales rep can review and send with one click.

**Configuration:**
- Operation: `Create Draft`
- To: `{{ $json.email_to }}`
- Subject: `{{ $json.email_subject }}`
- Message (HTML): `{{ $json.email_body }}`
- Additional Fields:
  - Reply To: _(leave blank or set to sales alias)_

**Credential:** Google OAuth2 — Gmail scope `gmail.compose`

**Notes:** This node runs on the High-fit path only (after Node 4). For Medium/Low leads, email drafting can be skipped or batched separately.

---

## Node 7 — Notify Sales via Slack

**Type:** Slack  
**Purpose:** Posts a summary card in a sales Slack channel so the team gets an instant heads-up on a new High-fit lead.

**Configuration:**
- Operation: `Post Message`
- Channel: `#sales-alerts` (or your channel)
- Text: _(leave blank — use blocks below)_
- Blocks (JSON):

```json
[
  {
    "type": "section",
    "text": {
      "type": "mrkdwn",
      "text": ":fire: *New High-Fit Lead*\n*{{ $json.full_name }}* — {{ $json.job_title }} at *{{ $json.company }}*"
    }
  },
  {
    "type": "section",
    "fields": [
      { "type": "mrkdwn", "text": "*Score:*\n{{ $json.lead_score }}" },
      { "type": "mrkdwn", "text": "*Timeline:*\n{{ $json.timeline }}" },
      { "type": "mrkdwn", "text": "*Budget:*\n{{ $json.budget_range }}" },
      { "type": "mrkdwn", "text": "*Source:*\n{{ $json.source }}" }
    ]
  },
  {
    "type": "section",
    "text": {
      "type": "mrkdwn",
      "text": "*Next Action:* {{ $json.next_action }}"
    }
  }
]
```

**Credential:** Slack OAuth2 — scope `chat:write`

## Connection Map

```
[1] Webhook
    └──→ [2] Format Lead Fields
              └──→ [3] IF: fit_level == High?
                        ├── TRUE  → [4] Sheets: High-Fit Queue
                        │               └──→ [5] Sheets: All Leads
                        │               └──→ [6] Gmail Draft
                        │               └──→ [7] Slack Notify
                        └── FALSE → [5] Sheets: All Leads
```

---

## Error Handling Recommendations

- Add a **Catch Error** node after Google Sheets and Gmail nodes to log failures without silently dropping leads.
- Use n8n's built-in **Retry on Fail** option (max 2 retries, 5s delay) on the Sheets append nodes.
- For production, add **Header Auth** to the Webhook node to prevent unauthorized submissions.
